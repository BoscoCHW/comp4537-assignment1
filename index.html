# Pokemon API ## Endpoints ### `GET /` - Description: Health check endpoint that returns `200 OK` status code - Response: `200 OK` ### `GET /pokemons` - Description: Returns a list of pokemons - Query parameters: - `count` (optional): Number of pokemons to return, default value is `10` - `after` (optional): Number of pokemons to skip, default value is `0` - Response: - `200 OK`: An array of pokemons - `500 Internal Server Error`: In case of any server error ### `POST /pokemon` - Description: Adds a new pokemon - Request body: - `id` (required): ID of the pokemon - `name` (required): Name of the pokemon - `type` (required): Type of the pokemon - `base` (required): Base stats of the pokemon - Response: - `200 OK`: JSON with `msg` key and the newly added pokemon - `400 Bad Request`: - If the pokemon already exists - If the request body is invalid - `500 Internal Server Error`: In case of any server error ### `GET /pokemon/:id` - Description: Returns a pokemon by ID - Path parameters: - `id`: ID of the pokemon - Response: - `200 OK`: JSON with pokemon information - `404 Not Found`: If the pokemon is not found - `400 Bad Request`: If the pokemon ID is not valid - `500 Internal Server Error`: In case of any server error ### `GET /pokemonImage/:id` - Description: Returns an image of the pokemon - Path parameters: - `id`: ID of the pokemon - Response: - `200 OK`: JSON with img key and the URL of the pokemon image - `404 Not Found`: If the pokemon is not found - `500 Internal Server Error`: In case of any server error ### `PUT /pokemon/:id` - Description: Updates an existing pokemon - Path parameters: - `id`: ID of the pokemon - Request body: - `name` (optional): New name of the pokemon - `img` (optional): New image URL of the pokemon - `type` (optional): New type of the pokemon - `base` (optional): New base stats of the pokemon - Response: - `200 OK`: JSON with msg key and the updated pokemon information - `404 Not Found`: If the pokemon is not found - `500 Internal Server Error`: In case of any server error ### `PATCH /pokemon/:id` - Description: Updates some fields of an existing pokemon - Path parameters: - `id`: ID of the pokemon - Request body: JSON object with the fields to update - Response: - `200 OK`: JSON with msg key and the updated pokemon information - `404 Not Found`: If the pokemon is not found - `400 Bad Request`: If the request body is invalid - `500 Internal Server Error`: In case of any server error ### `DELETE /pokemon/:id` - Description: Deletes an existing pokemon - Path parameters: - `id`: ID of the pokemon - Response: - `200 OK`: JSON with msg key and the deleted pokemon information - `404 Not Found`: If the pokemon is not found - `500 Internal Server Error`: In case of any server error
